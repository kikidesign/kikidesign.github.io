## Markdown 是什么？

<p>Markdown 是 2004 年由 John Gruber 设计和开发的纯文本格式的语法，是一种可以使用普通文本编辑器编写的标记语言，通过类似 HTML 的标记语法，它可以使普通文本内容具有一定的格式。所以通过同一个名字它可以使用工具来转换成 HTML。</p>
<p>Markdown 语法具有简单、易读易写的特点。</p>

<p>扩展阅读：</p>
<ul>

[Markdown - 维基百科，自由的百科全书]
</li>
<li>
[Markdown - Daringfireball]
</li>
</ul>

## 为什么写作者应当学习 Markdown？ 

<p>Markdown 的主要优势：</p>
<ul>
<li>轻量级，跨平台</li>
<li>语法简单，易读易写</li>
<li>干净，（呈现）一致性</li>
<li>可见即可得</li>
<li>快捷（专注文字内容的书写本身，而非排版样式）</li>
<li>抗干扰（语法代替排版，省却在键盘和工具栏之间来回切换，调整内容和格式的麻烦，一路都可以键盘搞定）</li>
<li>高兼容度（发布网页、制作电子书或其它）</li>
</ul>

<p>比较 word、pages、txt 或 HTML </p>
<p>对于多数人而言，比较核心的特点：</p>
<ul>
<li>人人能使用（排除了系统环境与样式差异的差异导致的干扰）</li>
<li>学习成本小（可以继续用自己熟悉的编辑器，只不过记得保存为文本格式）</li>
</ul>

## 如何令文字斜体显示？ 

<p>在 Markdown 中实现斜体的语法：</p>
<pre><code>    *斜体* 斜体
    _斜体_
</code></pre>
<p>使用星号「*」和下划线「_」来将文字转化为斜体</p>
<p>例子：</p>
<p>这是新中国历史上<em>最大</em>的一次诈骗案。</p>

## 如何对特定文字加粗? 

<p>加粗的语法很简单粗暴，在斜体语法的基础上，让星号或下划线符号多一个就行。</p>
<pre><code>    **加粗** 加粗
    __粗体__
</code></pre>
<p>使用双星号「**」和双下划线「__」来将文字进行强调。</p>
<p>例子：</p>
<p><strong>北京</strong>的<strong>冬天</strong>，自有它独特的美感。</p>
<p>说明：</p>
<p>对于强调符号的使用，需要注意的就是，以什么符号开始，就以什么符号结尾，数量保持一致，相应语法才能生效。</p>

## Markdown 分隔符怎么使用 

<p>实现分割线多样的语法形式：</p>
<pre><code>    *** 加分割线
    * * * 加分割线
    ***** 或者这样？
    --- 还有另外的方式
    ___ 其实，还可以这样
</code></pre>
<p>用的符号虽然不同（数量基本是一致的，3 个），效果都一样。</p>
<p>分割线用于实现区域的分隔，强化区块的差异。</p>

## 如何在 Markdown 中引用内容？ 

<p>引用：使用「&gt;」，添加在每行的开头。</p>
<p>例子：</p>
<pre><code>杜拉斯：
&gt; 愛之於我 不是肌膚之親 不是一蔬一飯。它是一種不死的慾望 是疲憊生活的英雄夢想。
</code></pre>
<p>两个注意点：</p>
<ol>
<li>有些编辑器会要求「&gt;」引用的上一行为空行,语法才能生效；</li>
<li>「&gt;」与其后引用的字之间添加一个空格。</li>
</ol>
<p>呈现效果：</p>
<p>杜拉斯：</p>
<blockquote>
  <p>愛之於我 不是肌膚之親 不是一蔬一飯。它是一種不死的慾望 是疲憊生活的英雄夢想。</p>
</blockquote>

## 在 Markdown 如何换行 

<p>说了这么多语法，咳，该如何在 Markdown 中正确换行呢？</p>
<p>换行使用的是这么两个方式：</p>
<ul>
<li>在行末敲两次回车键</li>
<li>在行末敲两个空格，一次回车键</li>
</ul>
<p>回车换行，在每行末尾，敲击两下回车键，隔开一个空行后继续书写；</p>
<p>运用空格换行比较适用于引用、代码区等场景。</p>

## 如何生成一个自动链接？ 

<p>自动链接或者说直接链接的实现语法是这样的：</p>
<pre><code>&lt;link&gt;  
</code></pre>
<p>通过尖角括号将链接包裹起来。</p>
<p>依然以 Google 主站为例：</p>
<pre><code>&lt;http://google.com&gt;
</code></pre>
<p>效果：</p>
<p>
[Google]
</p>
<p>还有一种链接是叫索引链接：</p>
<p><img class="full-image" src="http://openmindclub.qiniudn.com/Azeril/MarkdownIndexLinkAdd.png" alt="MarkdownIndexLinkAdd"></p>
<p>使用方括号作为引用链接。后一个方括号中可以添加数字、英文等字符，在文末将具体引用条目与对应注释内容进行关联。</p>
<p>功能是一致的，可点击，点击之后跳转到一个具体指向的网页链接。</p>

## 添加图片的常用方式 

<p>Markdown 添加图片的基本语法是这样的：</p>
<pre><code>![Pic name](Pic link)  
</code></pre>
<p>网址链接的语法前加「!」(英文感叹号)就是图片链接的形式了。当然，链接的网址必须是图片的分享链接才行。通常，我们称之为直链，也即图片直接链接。</p>
<p>Markdown 的日常使用，添加图片一般操作是先上传图片到某个图床，然后获取图片的外链再进行调用。</p>
<p>关于图床的知识，我们在后边会详细说到。</p>

## 如何在 Markdown 插入本地图片？ 

<p>添加本地图片的关键是路径。</p>
<pre><code>![image title](/Users/Usr_name/Downloads/hacker.jpg)
</code></pre>
<p>添加一张存放于本地 Downloads 目录下命名为 hacker 的 jpg 图片。</p>
<p>如果要将某篇含有图片的 Markdown 文章发给别人，图片存放于本地而非图床，则有必要将图片放置于文档相同的文件夹才行（当然，实际查看时因电脑系统的用户名不同，也需要做调整才能显示）。</p>
<p>本地添加图片的方式也适用于 GitHub 仓库托管图片的运用，包括 GitHub 博客内文中添加仓库托管图片时。</p>

## 如何用 Markdown 制作表格？ 

<pre><code>|Title |Title 1|Title 2|Title 3|
|---|---|---|---|
| A|B|C|D|
</code></pre>
<p>一般的表格由「|」与「-」两种符号（英文半角字符）构成。第 1、3及其后的行都由「|」加具体内容组成。依数据的列数确定数量（列数据量 +1）。 </p>
<p>第二行为中间为连续的「-」组成的隔断，数量至少 3 个，更多不限，更多可能是让文本显得美观（和预览无关）。</p>
<p>一个用 Markdown 语法制作的表格，呈现效果如下：</p>
<p>|Province | ZJ 浙江省 | FJ 福建省 | YN 云南省 |
|---|-----|-----|-----|
|省会|杭州|厦门|昆明|</p>

----------------

## 如何改变表格中文字对齐方式？ 

<p>文字 居左/中/右 显示的方法：</p>
<p>|Title |Title 1|Title 2|Title 3|
|---|---:|:---:|---:|
| A|B|C|D|</p>
<p>利用在「---」左右添加冒号「:」来实现排列方式的调整。</p>
<p>具体如下：</p>
<ol>
<li><code>:–--</code> 冒号在左，左对齐</li>
<li><code>--–：</code> 冒号在右，右对齐</li>
<li><code>：--–：</code> 左右两侧都出现冒号，居中对齐</li>
</ol>
<p>呈现效果如下：</p>
<p>|Province | ZJ 浙江省 | FJ 福建省 | YN 云南省 |
|---|:-----|:-----:|-----:|
|省会|杭州|厦门|昆明|</p>
<p>第二行中出现的冒号作用是设定表格内数据的对齐方式，不是必须使用的。</p>
